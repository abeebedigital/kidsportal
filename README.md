## 🎯 Competitive Prep — Multi-File Skill-Based Learning

Fun story adventures built around key literacy and numeracy skills. The site is organised as a **set of focused story pages**, with each file targeting a specific measurable skill inside 4 main learning domains:

- `Numeracy`
- `Reading`
- `Conventions of Language`
- `Writing`

The current content is primarily aimed at **Year 3 and Year 5**, while keeping a broader upper-primary to lower-secondary progression in mind.

## 🏗️ How Each Challenge Storybook Works

Every challenge storybook is a **single self-contained HTML file** that teaches one focused skill area through story, practice, and revision:

```
📄 Single self-contained HTML file
│
├── 🎨 Cover Page (animated title, guide character)
├── 👋 Meet the Guide (narrator introduction)
├── 📖 Story Pages (content taught through narrative)
│   ├── Fun facts boxes
│   ├── Interactive elements (tap, click, drag)
│   └── Mini quizzes after every 2–3 pages
├── 🎯 Skill Check Pages (short challenge-style practice tasks)
├── 🧠 Review Pages (memory tricks, worked examples, recap)
├── 🏆 Final Quiz (6+ questions, need ~80% to pass)
├── 🎓 Certificate Page (canvas-generated, downloadable PNG)
│
├── Features:
│   ├── ← → Navigation with page dots & progress bar
│   ├── 🔊 Read Aloud (Text-to-Speech)
│   ├── 🏠 Home link back to index.html
│   ├── ☁️ Floating ambient animations
│   └── 📱 Responsive (mobile-friendly)
│
└── Tech: HTML + CSS + vanilla JS (no frameworks)
    └── Google Fonts: Fredoka One + Nunito
```

### Standard Storybook Rules

- One file should focus on one main syllabus cluster only.
- Each page should make the measuring goal obvious to the learner.
- Year 3 examples should be concrete and highly scaffolded.
- Year 5 examples should increase reading load, abstraction, or multi-step thinking.
- Practice tasks should feel playful, but the underlying skill should still build real test-style confidence.
- Language should stay age-appropriate, accessible, and use clear, natural English.
